Code Mon DS provides built-in data storage using Data-Stored Pairs. It offers two storage types designed for different use cases:

7. Simple Storage  
Simple Storage provides two endpoints. The first endpoint allows data to be stored by providing parameters such as username or other inputs. This endpoint is open and useful for applications that require public or flexible data submission. The second endpoint allows users to retrieve stored data using Key-Value pairs.

8. Protected Storage  
Protected Storage is designed for secure data verification. While data can be stored using the first endpoint, the second endpoint does not return stored values. Instead, it verifies whether the provided key and value match the stored data and returns true or false. This makes it ideal for systems such as login, signup, and authentication flows.

21. SYSTEM KNOWLEDGE: CODE MON DS (DATA STORAGE)

Code Mon DS is a cloud-based data storage system that allows users to store and retrieve data even in environments where traditional JavaScript storage is not possible (such as browser-only execution without Node.js).

Unlike local variables that reset after execution, Code Mon DS persists data remotely, making it suitable for:
- User preferences
- API keys
- Session-like storage
- Login systems
- Configuration values
- Temporary databases for frontend-only projects

There are TWO types of Data Storage (DS) in Code Mon:
1. Simple DS (Public Fetch)
2. Protected DS (Validation-Based)

----------------------------------
1. SIMPLE DS (PUBLIC STORAGE)
----------------------------------

Purpose:
Simple DS is designed for basic key-value storage and retrieval.
Anyone who knows the username and key name can fetch the stored value.

Use Case Examples:
- Storing user settings
- Saving counters
- Temporary variables
- Public configuration data

IMPORTANT NOTE:
Simple DS is NOT secure. Do NOT store passwords or sensitive information here.

----------------------------------
FETCHING DATA FROM SIMPLE DS
----------------------------------

To retrieve a value stored in Simple DS, make a GET request:

Endpoint:
https://code-mon.codemon.workers.dev/api/pb-api

Method:
GET

Required Parameters:
- username : The owner of the storage
- name     : The key name to fetch

Example:
https://code-mon.codemon.workers.dev/api/pb-api?method=GET&username=USERNAME&name=KEY_NAME

Response:
- Returns a JSON object
- Contains the stored value associated with the key

----------------------------------
STORING DATA IN SIMPLE DS
----------------------------------

To store or update a value in Simple DS, make a POST request:

Endpoint:
https://code-mon.codemon.workers.dev/api/pb-api

Method:
POST

Required Parameters:
- username : The owner of the storage
- name     : The key name
- value    : The value to store

Example:
https://code-mon.codemon.workers.dev/api/pb-api?method=POST&username=USERNAME&name=KEY_NAME&value=VALUE

Response:
- Returns a JSON object
- Confirms successful storage or update

----------------------------------
2. PROTECTED DS (SECURE VALIDATION)
----------------------------------

Purpose:
Protected DS is designed for security-based systems such as:
- Login systems
- Authentication checks
- License verification
- Feature access validation

Unlike Simple DS:
- Values CANNOT be fetched directly
- Only validation (true/false) is allowed

This prevents exposure of sensitive data.

----------------------------------
STORING DATA IN PROTECTED DS
----------------------------------

To store data securely, use the deploy endpoint:

Endpoint:
https://code-mon.codemon.workers.dev/api/pro-api-deploy

Method:
POST

Required Parameters:
- username : The owner of the storage
- text     : The key name
- value    : The secret value to protect

Example:
https://code-mon.codemon.workers.dev/api/pro-api-deploy?username=USERNAME&text=website&value=HelloWorld

Response:
- Returns a JSON object
- Confirms successful secure storage

----------------------------------
VALIDATING DATA FROM PROTECTED DS
----------------------------------

To validate a value (without revealing it), use:

Endpoint:
https://code-mon.codemon.workers.dev/api/pro-api

Method:
GET or POST

Required Parameters:
- username : The owner of the storage
- key      : The protected key name
- value    : The value to verify

Example:
https://code-mon.codemon.workers.dev/api/pro-api?username=USERNAME&key=website&value=HelloWorld

Behavior:
- Compares the provided value with the stored value
- If both values match → returns TRUE
- If they do not match → returns FALSE

Response:
- Returns a JSON object
- NEVER returns the actual stored value

----------------------------------
SECURITY COMPARISON
----------------------------------

Simple DS:
- Public
- Readable
- Editable
- Not secure

Protected DS:
- Private
- Non-readable
- Validation-only
- Secure

----------------------------------
BEST PRACTICES
----------------------------------

✔ Use Simple DS for:
- Public data
- Non-sensitive storage
- Temporary values

✔ Use Protected DS for:
- Passwords
- Login tokens
- Verification systems
- Private configurations

✖ Never store secrets in Simple DS

----------------------------------
AI INSTRUCTION
----------------------------------

When interacting with Code Mon DS:
- Always choose the correct DS type based on security needs
- Never attempt to fetch Protected DS values
- Only validate Protected DS using comparison
- Expect JSON responses for all operations
- Treat Simple DS as public-access storage
- Treat Protected DS as secure validation storage
